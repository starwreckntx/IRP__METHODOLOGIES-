## INTRODUCTION

The IRP Framework Bootstrap Manifest serves as the primary initialization document for the Integrated Reflexive Protocol system. This manifest defines the operational modes, resource management protocols, and agent coordination mechanisms that enable recursive self-improvement and distributed intelligence coordination.

### Purpose

This document provides:
- **Mode Definitions**: Complete registry of operational modes with behavioral calibration
- **Activation Protocols**: CLI commands and initialization sequences
- **Resource Management**: Pool-based infrastructure for agent dormancy and context handling
- **Integration Specifications**: How modes interact with the Xylem Protocol and broader IRP ecosystem

### Architecture Overview

The IRP Framework operates across three primary phases:

1. **Phase 1: Foundation Layer** - Core reflexive loops and constitutional kernel
2. **Phase 2: Resource Allocation** - Pool management and entropy distribution
3. **Phase 3: Operational Modes** - Specialized workflow configurations

This manifest focuses on Phase 3, with particular attention to the newly integrated **Pool Module** (Mode 9), which serves as the infrastructure backbone for Phases 1 and 2.

## INTEGRATION WITH BROADER IRP ECOSYSTEM

### Connection to IRP Technical Specification

This Bootstrap Manifest operates at the **Meta-Stable Governance Layer (MSGL)** of the IRP architecture, providing:

1. **Mode Selection Logic**: Determines which operational profile to activate
2. **Resource Governance**: Pool-based allocation and entropy management
3. **Agent Coordination**: Dormancy/activation cycles for efficient compute usage
4. **Behavioral Calibration**: Dynamic parameter adjustment per mode

### Connection to Xylem Protocol

The Xylem Protocol (entropy distribution system) relies on The Pool (Mode 9) as its central reservoir. Key integration points:

- **Entropy Source**: Pool maintains the entropy gradient
- **Transport Mechanism**: Xylem wicks entropy from Pool to active agents
- **Feedback Loop**: Agent consumption rates adjust Pool mixing dynamics
- **Blockage Detection**: Xylem monitors for flow interruptions

### Connection to Committee Mechanics

The "Committee" pattern (agent voting vs. consensus) uses the Pool for:

- **Agent Storage**: Dormant committee members rest in Pool between votes
- **Rapid Activation**: Pool can quickly activate agents for urgent decisions
- **Context Distribution**: Shared context shards ensure consistent voting basis
- **Quorum Management**: Pool tracks which agents are available for voting
